OR-Set vs LWW-element-set

This page compares the same scripted history under tag-based observed-remove semantics and timestamp-based last-write-wins semantics. It is meant to make the conflict-resolution trade-off visible without reading raw JSON first.

The final states diverge on notebook: OR-Set leaves it present because the remove only tombstoned observed tags, while LWW leaves it absent after comparing add=1 and remove=4 under remove-wins ties.

  • 3 replicas (a, b, c)
  • 5 scripted steps
  • remove LWW tie bias
  • 1 final divergent element(s)

Why the models diverge

OR-Set: removes only the add-tags a replica has already observed, so a concurrent/new tag can survive later sync.
LWW-element-set: resolves add/remove conflicts by timestamp ordering; when timestamps tie, the configured bias decides the winner.

Final divergence notes

  • notebookOR-Set keeps only observed-remove tombstones, so active tags a:1 can survive. LWW compares add=1 vs remove=4 with remove-wins ties.
