# Project Selection Research

## Sources Reviewed
- GitHub/portfolio project recommendation roundups
- README best-practice guides
- testing best-practice guides
- Python standard-library CLI/database/http references
- Node CLI project patterns for fs/readline/http/JSON work

## Selection Principles
1. practical enough to discuss in interviews
2. small enough to finish thoroughly in one workspace
3. diverse across CS topics:
   - persistence: JSON, SQLite
   - algorithms: backtracking, graph search
   - systems: shell, file monitor, hashing
   - backend: HTTP APIs, static site generation
   - tooling: CLIs, file organization, report generation
4. no fragile external dependencies required for the default happy path
5. clear room for v2 improvements

## Expected Portfolio Features Incorporated
- crisp problem statement per project
- readable code split into modules
- tests for core behavior
- runnable demo commands
- README with setup, usage, architecture, and future improvements
- error handling and input validation
