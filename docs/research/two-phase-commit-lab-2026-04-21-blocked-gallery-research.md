# Two-phase commit lab research note — 2026-04-21 — blocked timeline gallery slice

## Goal
Add a compact cross-scenario gallery page that lines up the blocked timeline PNG covers and makes the full HTML/SVG/Markdown walkthrough artifacts easy to reach from one recruiter-friendly page.

## Brief research summary
- The project already had the hardest prerequisite in place: deterministic blocked-case PNG covers generated by the catalog flow, so this slice could stay dependency-free and reuse the existing artifact set instead of introducing a new renderer.
- The existing incident-response dashboard already grouped blocked cases by response story, so the new page should complement it instead of duplicating it. The dashboard stays triage-first; the gallery should be screenshot-first.
- The most useful gallery interaction is not opening the raw PNG itself, but using the cover as the visual entry point into the richer timeline HTML/SVG walkthroughs.

## Scope chosen for this slice
- generate a dedicated blocked timeline gallery HTML page during `catalog`
- add direct navigation links between the catalog, incident-response dashboard, and blocked gallery
- keep the page self-contained and static so the committed artifact works on GitHub Pages and in local file browsing

## Sources used
- existing `two_phase_commit_lab.py` catalog/dashboard generation patterns in this repo
- existing committed blocked timeline PNG artifacts under `docs/artifacts/two-phase-commit-lab/`
